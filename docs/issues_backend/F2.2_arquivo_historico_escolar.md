## Contexto

O sistema foi implantado em 2026, mas o Colegio Conexao Maranhense existe desde 2020. Alunos que estudaram na escola em anos anteriores nao tem registros no novo sistema. Para emitir um historico escolar completo e retroativo, e necessario digitalizar os documentos em papel usando IA.

## O problema

Uma aluna que estudou na escola em 2024 (2 serie) e 2025 (3 serie) nao pode ter seu historico gerado pelo sistema atual porque esses registros nao existem no banco. O mesmo vale para alunos que se formaram ou se transferiram nos ultimos 5 anos.

## Solucao proposta

Fluxo de 3 etapas na mesma tela, usando a Edge Function `claude-proxy` ja existente:

**Etapa 1 - Digitalizar ficha de matricula**
Upload de PDF/foto da ficha de matricula antiga.
IA extrai: nome, CPF, data nascimento, filiacao, serie, ano, segmento.
Secretaria revisa e salva - cria perfil em `alunos_historicos`.

**Etapa 2 - Digitalizar boletins da Conexao**
Upload de PDF/foto do boletim (pode ser por ano ou por bimestre).
IA extrai: disciplinas, notas por bimestre, medias, faltas, ano letivo, serie.
Secretaria revisa e salva em `boletins_historicos`.

**Etapa 3 - Historico da escola anterior**
Ja existe via `HistoricoIA.tsx` + tabela `historico_externo`.
Adaptar para vincular a `alunos_historicos` (nao so a `users`).

**Gerar historico retroativo**
Botao que combina os 3 tipos de dado e gera PDF completo via `EmissaoDocumentos`.

## Novas tabelas

### `alunos_historicos`

```sql
CREATE TABLE alunos_historicos (
  id                uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  nome              text NOT NULL,
  data_nascimento   date,
  cpf               text,
  filiacao          text,
  serie_saida       text,
  ano_saida         integer,
  motivo_saida      text CHECK (motivo_saida IN ('conclusao','transferencia','desistencia','outro')),
  segmento          text NOT NULL DEFAULT 'ead',
  arquivo_ficha_url text,
  criado_em         timestamptz DEFAULT now(),
  criado_por        uuid REFERENCES users(id)
);
```

### `boletins_historicos`

```sql
CREATE TABLE boletins_historicos (
  id                  uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  aluno_id            uuid NOT NULL REFERENCES alunos_historicos(id) ON DELETE CASCADE,
  disciplina          text NOT NULL,
  serie               text NOT NULL,
  ano_letivo          integer NOT NULL,
  bimestre            integer NOT NULL CHECK (bimestre BETWEEN 1 AND 4),
  av1                 numeric,
  av2                 numeric,
  av3                 numeric,
  recuperacao         numeric,
  media_final         numeric,
  faltas              integer DEFAULT 0,
  carga_horaria       integer DEFAULT 0,
  situacao            text CHECK (situacao IN ('aprovado','reprovado','recuperacao','cursando')),
  segmento            text NOT NULL DEFAULT 'ead',
  arquivo_boletim_url text,
  criado_em           timestamptz DEFAULT now(),
  criado_por          uuid REFERENCES users(id)
);
```

## Adaptacao de `historico_externo`

Adicionar coluna `aluno_historico_id` como alternativa ao `aluno_id` (users):

```sql
ALTER TABLE historico_externo
  ADD COLUMN aluno_historico_id uuid REFERENCES alunos_historicos(id);
-- aluno_id continua para usuarios ativos no sistema
-- aluno_historico_id para alunos historicos sem conta
```

## Novo componente

`src/components/ArquivoHistorico.tsx` - fluxo de 3 etapas:

1. Busca/criacao do perfil do aluno historico
2. Upload e digitalizacao de boletins da Conexao
3. Upload do historico da escola anterior (reutiliza HistoricoIA)
4. Geracao do historico retroativo completo

## Integracao

Adicionar no menu da Secretaria e Gestor Geral: "Arquivo Historico".

## Risco

Zero para producao - apenas novas tabelas e novo componente.
Os dados ativos (users, notas, fichas_matricula) nao sao tocados.
