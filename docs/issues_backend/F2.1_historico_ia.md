## Contexto

O `HistoricoIA.tsx` ja existe com UI completa (upload de PDF/foto, extracao de dados, revisao, salvar), mas esta completamente quebrado:

- Chama `https://api.anthropic.com/v1/messages` **diretamente do front-end** sem chave de API — sempre falha com 401
- Salva em tabela `historico_externo` que **nao existe** no banco
- **Nao esta integrado** em nenhuma rota/menu do sistema

## O que este issue entrega

O fluxo "60 dias para segundos": secretaria recebe historico em papel de outra escola, faz upload do PDF/foto, Claude extrai todas as disciplinas/notas/dados em segundos, secretaria revisa e confirma.

## Tarefas

### 1. SQL — tabela `historico_externo` + RLS (Supabase Dashboard)

```sql
CREATE TABLE historico_externo (
  id                   uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  aluno_id             uuid REFERENCES users(id),
  nome_aluno           text NOT NULL,
  data_nascimento      date,
  nome_escola_anterior text NOT NULL,
  municipio_escola     text,
  uf_escola            text,
  cnpj_escola          text,
  nivel_ensino         text CHECK (nivel_ensino IN ('fundamental','medio')),
  disciplinas          jsonb NOT NULL DEFAULT '[]',
  observacoes          text,
  arquivo_url          text,
  criado_por           uuid REFERENCES users(id),
  segmento             text NOT NULL DEFAULT 'ead',
  criado_em            timestamptz DEFAULT now()
);

ALTER TABLE historico_externo ENABLE ROW LEVEL SECURITY;

CREATE POLICY "he_admin"
  ON historico_externo FOR ALL USING (eh_admin());

CREATE POLICY "he_insert_staff"
  ON historico_externo FOR INSERT
  WITH CHECK (tem_tipo(ARRAY['gestor_geral','secretaria','coordenador']) AND segmento = get_segmento_usuario());

CREATE POLICY "he_select_staff"
  ON historico_externo FOR SELECT
  USING (tem_tipo(ARRAY['gestor_geral','secretaria','coordenador']) AND segmento = get_segmento_usuario());

CREATE POLICY "he_select_aluno"
  ON historico_externo FOR SELECT
  USING (tem_tipo(ARRAY['aluno']) AND aluno_id = auth.uid());
```

### 2. Edge Function `claude-proxy` (nova — Supabase Functions)

Arquivo: `supabase/functions/claude-proxy/index.ts`

- Recebe `{ conteudo_base64, media_type, is_pdf, prompt }` do front-end
- Usa secret `ANTHROPIC_API_KEY` (configurado em Supabase > Settings > Edge Functions > Secrets)
- Chama a Anthropic API server-side com o modelo `claude-opus-4-5`
- Retorna `{ texto }` com o JSON extraido
- Custo estimado: menos de R$ 0,10 por historico

**Setup unico necessario:** apos deploy da Edge Function, adicionar em Supabase Dashboard > Settings > Edge Functions > Secrets:
`ANTHROPIC_API_KEY = sk-ant-...`

### 3. Corrigir `HistoricoIA.tsx`

Substituir o `fetch` direto para a Anthropic API pela chamada a Edge Function:

```typescript
// ANTES (inseguro — expoe API key se tiver):
const response = await fetch('https://api.anthropic.com/v1/messages', { ... })

// DEPOIS (seguro — chave fica no servidor):
const { data, error } = await supabase.functions.invoke('claude-proxy', {
  body: { conteudo_base64: base64, media_type: arquivo.type, is_pdf: isPdf, prompt }
})
```

Adicionar `HistoricoIA` como aba no menu de Documentos da secretaria/coordenador.
