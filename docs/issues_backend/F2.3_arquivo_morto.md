## Contexto

O fluxo de digitalizacao (F2.2) ja funciona, mas falta:
- Visao de gerenciamento: quais alunos tem ficha, boletins e historico externo digitalizado
- Impressao do historico retroativo completo
- Controle de exclusao segura de registros incorretos
- Ponto de entrada unificado no menu ("Arquivo Morto")

## O que este issue entrega

### Menu lateral
Novo item "Arquivo Morto" (substitui "Arquivo Historico") com 2 abas:
- Gerenciamento: lista alunos historicos com status de digitalizacao
- Digitalizar: fluxo existente (ArquivoHistorico.tsx)

### Aba Gerenciamento
Tabela com colunas:
- Nome | Serie/Ano Saida | Motivo | Ficha (check/x) | Boletins (N disc.) | Hist. Ext. (check/x) | Acoes

Acoes por linha:
- [Imprimir Historico] — gera PDF retroativo completo
- [+ Dados] — abre aba Digitalizar pre-selecionando o aluno
- [Deletar] — SOMENTE admin/gestor_geral, com confirmacao

### Impressao do historico retroativo
Documento HTML (mesmo padrao do EmissaoDocumentos) combinando:
1. Dados do aluno (alunos_historicos)
2. Historico escola anterior (historico_externo)
3. Boletins da Conexao por ano (boletins_historicos)

### Delete seguro (soft delete)
- Coluna deletado_em + deletado_por em alunos_historicos
- Nunca apaga fisicamente — apenas marca como deletado
- Visivel apenas para administrador e gestor_geral
- AlertDialog com confirmacao: digitar o nome do aluno
- CASCADE: boletins e historico externo ficam no banco (orfaos, mas recuperaveis)

## SQL necessario

```sql
ALTER TABLE alunos_historicos
  ADD COLUMN IF NOT EXISTS deletado_em timestamptz,
  ADD COLUMN IF NOT EXISTS deletado_por uuid REFERENCES users(id);

CREATE INDEX IF NOT EXISTS idx_ah_deletado ON alunos_historicos(deletado_em)
  WHERE deletado_em IS NULL;
```
